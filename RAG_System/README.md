## Актуальность

В корпоративной среде объемы документов огромны. RAG-система — это AI-агент, который быстро находит релевантную информацию, экономя время руководителей, инженеров и менеджеров.  
Забыли название файла или где искать ответ? Система вспомнит и предоставит точный результат.
Ежедневно мы сталкиваемся с вызовами: огромные архивы документов, забытые названия файлов, потерянные детали в отчетах.  
Необходимость быстрого поиска ответов без глубокого погружения в каждый файл.  
RAG-система решает эти проблемы, используя AI для мгновенного извлечения данных, повышая эффективность команд.  

- База данных с документами слишком большая?  
- Забыли название документа или исследования?  
- Необходимо точно и быстро достать нужную информацию?  

AI-Агент поможет вам решить ваши бюрократические проблемы.  

## Компоненты RAG-системы

1. Хранение данных  
Чанки (кусочки) документов хранятся в векторной БД в текстовом и математическом виде (PostgreSQL + pgvector)  
2. Поиск  
Запрос переводится на язык математики и с помощью косинусного сходства из БД достаются схожие документы  
3. Генерация  
LLM генерирует ответ на основе вашего запроса и поданного из БД контекста.  

## Выбор эмбеддинговой модели
Эксперименты включали русские модели (ruBert) и мультиязычные (multilingual-e5).  
Оценивались модели по точности поиска на корпоративном датасете с русским текстом.  
Multilingual-e5 лидирует: топ-10 в MTEB (мировое соревнование по переводу) по семантическому представлению русского языка, небольшой размер и высокая скорость.

## Реализованные фичи

- Контекст в чате: Множественные вопросы по одному документу. Модель помнит весь контекст  
- Источники: Предоставление файлов-источников, которые сразу можно скачать и проверить  
- Удобный интерфейс: Интуитивный дизайн + подробная инструкция для быстрого старта
- Логирование запросов: Сохранение запросов в БД для анализа использования агента и частых ошибок  

## Точки роста проекта

1. Масштабирование (через сервер)  
2. Мультимодальный анализ (обработка изображений, формул, таблиц)  
3. Авторизация и кастомизация (доступ только к разрешенным документам, расширенные права и настраивание системы под свои нужды)  
4. Редактирование документов (создание и редактирование документов через агента при соответствующих правах)
